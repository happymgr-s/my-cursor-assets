# オンボーディングフローテンプレート

ユーザーが素早く成功できるように、明確で励ますオンボーディングエクスペリエンスを設計するためにこのテンプレートを使用します。

## オンボーディングの原則

1. **早期に価値を示す** — 最初のセッションでユーザーが成功できるように支援
2. **段階的な開示** — すべての機能を一度に見せない
3. **可能な限りオプション** — ユーザーがスキップして探索できるようにする
4. **小さな成功を祝う** — 完了した各ステップを認識
5. **簡潔に** — ユーザーはプロダクトについて読むのではなく、使い始めたい

## コアフロー構造

```
ウェルカム → セットアップ → 最初の成功 → 次のステップ
```

各ステップは以下に答えるべき：
- **どこにいるか？**（進捗インジケーター）
- **ここで何をするか？**（明確な指示）
- **なぜこれが重要か？**（ユーザーへのメリット）
- **スキップできるか？**（終了オプション）

## テンプレート: ウェルカム画面

**目的**: ユーザーを方向づけ、期待を設定

```
**見出し**: [ウェルカム + 価値提案]
**本文**: [2〜3のメリット、箇条書き]
**主要CTA**: [開始アクション]
**副次CTA**: [サインイン（該当する場合）]
```

**例:**
```
**見出し**: TaskFlowへようこそ
**本文**: 
• チームでプロジェクトを整理
• リアルタイムで進捗を追跡
• 締め切りを逃さない

**主要CTA**: アカウントを作成
**副次CTA**: サインイン
```

## テンプレート: アカウントセットアップ

**目的**: 必須情報のみを収集

```
**タイトル**: [アクション指向のタイトル]
**本文**: [この情報がなぜ必要か]
**フィールド**: [最小限の必須フィールド]
**主要CTA**: [続行/次のアクション]
**副次CTA**: [スキップ（本当にオプションの場合）]
```

**例:**
```
**タイトル**: あなたについて教えてください
**本文**: エクスペリエンスをパーソナライズします。

**フィールド**:
- 名前
- メール
- パスワード

**主要CTA**: 続行
**副次CTA**: 後で行う
```

**ベストプラクティス:**
- 今絶対に必要なもののみを尋ねる
- 各フィールドがなぜ必要かを説明
- 可能な場合はスマートデフォルトを使用
- オプションのステップはスキップを許可

---

## テンプレート: 機能紹介

**目的**: アクションを通じて一度に1つの機能を教える

```
**タイトル**: [機能名 + メリット]
**本文**: [簡潔な説明、1〜2文]
**ビジュアル**: [スクリーンショットまたはイラスト]
**主要CTA**: [機能を試すアクション]
**副次CTA**: スキップ / 次へ
**進捗**: [Y個中X個のステップ]
```

**例:**
```
**タイトル**: 最初のプロジェクトを作成
**本文**: プロジェクトは関連するタスクを整理し、チームと協力するのに役立ちます。

[ビジュアル: プロジェクトビューのスクリーンショット]

**主要CTA**: プロジェクトを作成
**副次CTA**: 今はスキップ
**進捗**: ステップ2/4
```

**ベストプラクティス:**
- 説明ではなく表示（ビジュアルを使用）
- ユーザーがすぐに試せるようにする
- 説明を20語以下に保つ
- 常にスキップを許可

---

## テンプレート: 最初の成功 / 完了

**目的**: ユーザーの最初の成功を祝い、次のアクションを促す

```
**タイトル**: [祝福のステートメント]
**本文**: [達成したこと + 次に可能なこと]
**主要CTA**: [次の自然なアクション]
**副次CTA**: [代替またはプロダクトへの退出]
```

**例:**
```
**タイトル**: すべて設定完了です！
**本文**: 最初のプロジェクトを作成しました。チームを招待する準備はできていますか？

**主要CTA**: チームメンバーを招待
**副次CTA**: 自分で探索
```

**ベストプラクティス:**
- 感嘆符を控えめに使用（フローごとに最大1回）
- 祝福が獲得されたものに感じられるようにする
- 論理的な次のステップを提案
- メインプロダクトへの退出を提供

## オンボーディングステップチェックリスト

オンボーディングフローの各ステップについて確認：

- [ ] **単一の焦点** — ステップごとに1つのアクションまたはコンセプト
- [ ] **明確な価値** — ユーザーはなぜこれが重要かを知っている
- [ ] **実行可能** — ユーザーは何をすべきか正確に知っている
- [ ] **スキップ可能** — オプションのステップはスキップできる
- [ ] **進捗が見える** — ユーザーは残りのステップ数を知っている
- [ ] **簡潔** — 本文テキストは30語以下
- [ ] **適切なトーン** — 上から目線でなく励ます

## ステップタイプ別のコンテンツパターン

### ウェルカム/紹介
**パターン**: 「[プロダクト]へようこそ。[Yを達成するためにXを行う]。」

例:
- 「Notionへようこそ。美しいドキュメント、Wiki、プロジェクトを作成できます。」
- 「Figmaへようこそ。リアルタイムでデザイン、プロトタイプ、コラボレーションできます。」

### 権限リクエスト
**パターン**: 「[具体的なメリット]のために[権限]を有効にする」

例:
- 「チームアクティビティの最新情報を受け取るために通知を有効にする」
- 「ドキュメントを即座にスキャンするためにカメラアクセスを許可する」

### 機能チュートリアル
**パターン**: 「[アクション]。[簡潔なメリット]。」

例:
- 「最初のタスクを追加。整理して締め切りを逃さない。」
- 「ワークスペースを作成。チームと1か所でコラボレーション。」

### 完了
**パターン**: 「[祝福]！[今可能なこと]。[オプション: 次のステップ]」

例:
- 「準備完了です！作成を開始するか、チームメイトを招待してコラボレーションしましょう。」
- 「セットアップ完了！ワークスペースがチームの準備ができました。」

## トーンバリエーション

### プロフェッショナル/B2B
```
**タイトル**: ワークスペースをセットアップ
**本文**: チームメンバーを追加して最初のプロジェクトを作成します。
**CTA**: セットアップを続行
```

### コンシューマー/カジュアル
```
**タイトル**: 始めましょう！
**本文**: これは1分しかかかりません。
**CTA**: 始める
```

### 技術的/開発者
```
**タイトル**: 環境を設定
**本文**: リポジトリを接続してデプロイを開始します。
**CTA**: リポジトリを接続
```

## 複数画面フローの例

### 画面1: ウェルカム
```
**WriteRightへようこそ**

AI駆動の編集でより良く、より速く書く。

• 文法を即座に修正
• 明確さとトーンを改善
• 自信を持って書く

[始める]
```

### 画面2: セットアップ
```
**WriteRightを使う目的は何ですか？** (4個中2個)

該当するものをすべて選択：
□ ビジネスライティング
□ クリエイティブライティング
□ 学生論文
□ 個人プロジェクト

[続行] [スキップ]
```

### 画面3: 最初のアクション
```
**試してみましょう** (4個中3個)

以下に任意のテキストを貼り付けて、WriteRightがそれを改善するのを見てください。

[テキスト入力ボックス]

[私のライティングを分析] [スキップ]
```

### 画面4: 成功
```
**よくできました！**

最初の文章を改善しました。次の傑作を書く準備はできていますか？

[書き始める] [ヒントを見る]
```

## 避けるべき一般的な間違い

❌ **ステップが多すぎる**: プロダクトを使用する前に10画面以上
✅ **焦点を絞ったフロー**: 最初の価値まで3〜5ステップ

❌ **機能の羅列**: 「できることが47個あります！」
✅ **段階的な開示**: 一度に1つの主要機能

❌ **スキップオプションなし**: すべての画面を強制的に通過させる
✅ **選択を尊重**: オプション機能にはスキップが利用可能

❌ **受動態**: 「アカウントが作成されました」
✅ **能動態**: 「アカウントを作成しました！」

❌ **企業用語**: 「生産性の向上を促進」
✅ **平易な言葉**: 「より多くのことを成し遂げる」

**価値を示す最初のユーザーアクション:**
**祝福/成功の瞬間:**

**下書きフロー:**

## オンボーディングのテスト

答えるべき質問:
1. ユーザーは2分以内にオンボーディングを完了できるか？
2. ユーザーは最初のセッションで意味のあることを達成できるか？
3. ユーザーは混乱せずにオプションのステップをスキップできるか？
4. トーンは上から目線でなく励ますものか？
5. 各画面には単一の明確な目的があるか？
