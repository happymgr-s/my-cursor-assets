# エラーメッセージテンプレート

ユーザーが回復できるように、明確で実行可能なエラーメッセージを書くためにこのテンプレートを使用します。

## 構造

```
[失敗したこと] [失敗した理由（わかる場合）] [次にすべきこと]
```

## テンプレート

### インラインエラー（フォームバリデーション）

**形式**: 簡潔で即座の修正ガイダンス

```
[フィールドの要件または制約]
```

**例:**

- メールアドレスに@を含めてください
- パスワードは 8 文字以上必要です
- カード番号が不完全です
- 未来の日付を選択してください

---

### 迂回エラー（回復可能な問題）

**形式**: 問題 + 解決策

```
**タイトル**: [失敗したアクション]
**本文**: [簡潔な説明]。[回復手順]。
**ボタン**: [具体的な回復アクション]
```

**例:**

```
**タイトル**: 変更を保存できません
**本文**: インターネット接続を確認してもう一度お試しください。
**ボタン**: 再試行
```

---

### ブロッキングエラー（システム問題）

**形式**: 明確な説明 + タイムライン + 安心感

```
**タイトル**: [利用できないもの]
**本文**: [利用できない理由]。[いつ利用可能になるか]。[ユーザーデータについての安心感]。
**ボタン**: [ステータス確認または代替アクション]
```

**例:**

```
**タイトル**: サービスが一時的に利用できません
**本文**: システムを更新中で、約15分後に復旧します。データは安全です。
**ボタン**: ステータスを確認
```

---

## エラーメッセージチェックリスト

エラーメッセージを確定する前に確認：

- [ ] **非難を避ける** — 「無効」「不正」「間違い」「エラー」なし
- [ ] **共感的なトーン** — ユーザーのフラストレーションを認識
- [ ] **具体的な問題** — 一般的な「問題が発生しました」ではない
- [ ] **明確な回復** — ユーザーに何をすべきか正確に伝える
- [ ] **前置き** — 最も重要な情報を最初に
- [ ] **能動態** — 「保存できませんでした」であって「変更を保存できませんでした」ではない
- [ ] **人間の言葉** — システムコードや技術的な専門用語ではない

## コンテキスト別のボイスバリエーション

### 重要度の高いエラー（支払い、セキュリティ、データ損失）

**トーン**: 真剣、明確、安心させる

```
支払いを処理できませんでした。カードは請求されていません。カード情報を確認してもう一度お試しください。
```

### 重要度の低いエラー（オプション機能、あると便利な機能）

**トーン**: 軽い、役立つ、劇的でない

```
プレビューを読み込めませんでした。更新してもう一度お試しください。
```

### 初回ユーザーエラー

**トーン**: 教育的、忍耐強い

```
プロフィール写真は5MB以下である必要があります。小さいファイルを試すか、画像を圧縮してください。
```

## 避けるべき一般的な間違い

❌ **曖昧**: 「エラーが発生しました」
✅ **具体的**: 「変更を保存できませんでした」

❌ **非難**: 「無効なメールアドレス」
✅ **ガイド**: 「メールアドレスに@を含めてください」

❌ **技術的**: 「ERR_CONNECTION_TIMEOUT」
✅ **人間的**: 「接続がタイムアウトしました。インターネットを確認してもう一度お試しください。」

❌ **解決策なし**: 「アップロードに失敗しました」
✅ **実行可能**: 「アップロードに失敗しました。ファイルサイズを確認してもう一度お試しください。」

❌ **受動態**: 「リクエストを受け付けられません。」
✅ **能動態**: 「リクエストを処理できませんでした」

## クイック記入テンプレート

迅速なエラーメッセージ作成にこれを使用：

**失敗したこと:**
**理由（わかる場合）:**
**ユーザーがすべきこと:**

**下書き:**
[失敗したこと]。[理由（わかる場合）]。[次のアクション]。

**記入例:**

- 失敗したこと: 招待を送信できませんでした
- 理由: メールがバウンスした
- すべきこと: スペルを確認

下書き: 「招待を送信できませんでした。メールアドレスを確認してもう一度お試しください。」
