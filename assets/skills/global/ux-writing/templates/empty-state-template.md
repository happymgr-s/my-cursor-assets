# 空状態テンプレート

コンテンツがないときにユーザーをガイドし、スペースを埋めるアクションを起こさせるためにこのテンプレートを使用します。

## 構造

```
[空状態を説明するタイトル] [簡潔な説明または励まし] [明確な行動喚起]
```

## テンプレートタイプ

### 初回使用の空状態
**目的**: 新規ユーザーにコンテンツを入力するようガイド
**トーン**: 招待的、励ます、メリットが明確

```
**タイトル**: [空のもの]
**本文**: [コンテンツを追加するメリットの簡潔な説明]
**ボタン**: [入力するための具体的なアクション]
```

**例:**
```
**タイトル**: まだプロジェクトがありません
**本文**: 最初のプロジェクトを作成して作業の整理を始めましょう。
**ボタン**: プロジェクトを作成
```

---

### ユーザーがクリアした空状態
**目的**: 完了を確認、ポジティブな強化
**トーン**: ポジティブ、祝福的（適切に）

```
**タイトル**: [ポジティブな完了ステートメント]
**本文**: [オプション: 次に何が起こるか、新しいコンテンツがいつ表示されるか]
```

**例:**
```
**タイトル**: すべて完了しました
**本文**: 新しいタスクが割り当てられるとここに表示されます。
```

---

### エラー/結果なしの空状態
**目的**: 検索/フィルターが何も返さないときに代替案を提案
**トーン**: 役立つ、解決策重視

```
**タイトル**: [見つからなかったもの]
**本文**: [変更の提案または代替アクション]
**ボタン**: [代替アクション]
```

**例:**
```
**タイトル**: 「ビンテージカメラ」の結果がありません
**本文**: 別のキーワードを試すか、すべてのアイテムを閲覧してください。
**ボタン**: すべてのアイテムを閲覧
```

---

## 空状態チェックリスト

空状態を確定する前に確認：

- [ ] **明確なコンテキスト** — ユーザーは何が空で、なぜ空かを知っている
- [ ] **実行可能** — 具体的な次のステップを提供
- [ ] **適切なトーン** — 感情的なコンテキストに合っている
- [ ] **価値重視** — アクションを起こすメリットを示す
- [ ] **簡潔** — すぐにスキャンできるほど簡潔
- [ ] **ネガティブを避ける** — 「何もありません」や「〜がありません」なし

## タイプ別のコンテンツパターン

### 初回使用（オンボーディング）
**パターン**: 「まだ[コンテンツ]がありません。[メリットステートメント]。[アクション]」

例:
- 「まだ連絡先がありません。連絡先をインポートしてメッセージを開始しましょう。**連絡先をインポート**」
- 「お気に入りが保存されていません。アイテムを保存して後で素早く見つけられます。**アイテムを閲覧**」
- 「まだチームメンバーがいません。プロジェクトで協力する人を招待しましょう。**チームを招待**」

### ユーザーがクリア（完了）
**パターン**: 「すべて[完了/完全/追いついた]。[オプション: いつもっと表示されるか]」

例:
- 「すべてのタスクが完了しました。チームメイトがタスクを割り当てると新しいタスクが表示されます。」
- 「受信トレイゼロ！すべてのメッセージを読みました。」
- 「すべて設定完了です。明日新しいおすすめをチェックしてください。」

### 結果なし（検索/フィルター）
**パターン**: 「[コンテンツ]が[条件]に一致しません。[提案]。[代替案]」

例:
- 「検索に一致するファイルがありません。別のキーワードを試すか、**すべてのファイルを表示**してください。」
- 「今月のイベントがありません。**今後のイベントを見る**か、**イベントを作成**してください。」
- 「これらのフィルターに一致するチームメンバーがいません。**フィルターをクリア**してすべてのメンバーを表示してください。」

### 権限/アクセスが必要
**パターン**: 「[コンテンツ]は利用できません。[理由]。[アクセスを取得する方法]」

例:
- 「レポートはまだ利用できません。詳細な分析にアクセスするにはプレミアムにアップグレードしてください。**プランを表示**」
- 「このフォルダは非公開です。所有者に共有を依頼してください。」
- 「カレンダーイベントは非表示です。イベントを表示するには**設定**でカレンダー同期を有効にしてください。」

## ボイスバリエーション

### プロフェッショナル/B2B製品
```
**タイトル**: ドキュメントがアップロードされていません
**本文**: チームと共有するファイルをアップロードしてください。
**ボタン**: ドキュメントをアップロード
```

### コンシューマー/フレンドリー製品
```
**タイトル**: まだ何もありません！
**本文**: お気に入りを追加してコレクションを構築し始めましょう。
**ボタン**: お気に入りを見つける
```

### 真剣/重要度の高い製品
```
**タイトル**: アラートなし
**本文**: システムは正常に動作しています。
```

## イラストレーションガイダンス

デザイナーと空状態のイラストレーションに取り組むとき：

**初回使用**: 楽観的で招待的なイメージ（開いた箱、白いキャンバス、スタート地点）
**ユーザーがクリア**: ポジティブで完了のイメージ（チェックマーク、きれいなスペース、祝福）
**結果なし**: 中立的で役立つイメージ（検索アイコン、虫眼鏡、方向矢印）

## 避けるべき一般的な間違い

❌ **ネガティブなフレーミング**: 「連絡先がありません」
✅ **中立/ポジティブ**: 「まだ連絡先がありません」

❌ **ガイダンスなし**: 「空」
✅ **実行可能**: 「まだ連絡先がありません。インポートして始めましょう。」

❌ **複数のCTA**: 3つの異なるボタンが競合
✅ **単一の主要CTA**: 1つの明確な次のアクション

❌ **テキストが多すぎる**: 機能を説明する長い段落
✅ **簡潔**: 1文のメリット、明確なボタン

❌ **技術的**: 「データベースにレコードが見つかりません」
✅ **人間的**: 「プロジェクトが見つかりません」

## クイック記入テンプレート

迅速な空状態作成にこれを使用：

**空状態タイプ:** (初回使用 / ユーザーがクリア / 結果なし / 権限)
**空のもの:**
**なぜ空か:**
**ユーザーがすべきこと:**
**ユーザーのメリット:**

**下書き:**
**タイトル**: [空のもの]
**本文**: [アクションのメリットまたは次に何が起こるか]
**ボタン**: [具体的なアクション]

**記入例:**
- タイプ: 初回使用
- 空のもの: 保存されたレポート
- 理由: 新規ユーザー
- すべきこと: レポートを作成または保存
- メリット: 重要なデータへの素早いアクセス

下書き:
**タイトル**: まだ保存されたレポートがありません
**本文**: レポートを保存して最も重要なデータに素早くアクセスできます。
**ボタン**: レポートを作成
