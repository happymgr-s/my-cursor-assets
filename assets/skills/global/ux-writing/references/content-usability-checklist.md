# コンテンツユーザビリティチェックリスト

UXテキストの品質を評価するためにこのチェックリストを使用します。各基準を0〜10で評価します。

## 簡潔性

**すべての言葉に明確な役割がある**
- 「基本的に」「実際に」「ただ」のようなフィラーワードなし
- 各言葉が意味または明確さを追加
- 意味を失わずに言葉を削除できない

**最小限の言葉で高い情報密度**
- 言葉あたりの最大の意味
- 効率的な表現
- 冗長性なし

**1行あたり40〜60文字**
- 可読性のための最適な行の長さ
- スキャンのために適切に分割
- 長すぎない（集中力の喪失を引き起こす）または短すぎない（前後の目の動きを引き起こす）

**短い文と段落**
- 段落あたり3〜4行未満
- 文の長さは変化するが平均約15〜20語
- スキャン可能なチャンクに分割

**シグナルワードで前置き**
- 最も重要な言葉が最初に来る
- 文の最初にアクションワード
- ユーザーはすぐに重要な情報を見る

**優先順位でアイデアを整理**
- 最も重要な情報が最初
- 二次的な詳細が続く
- 知っておくと良い情報が最後

## 目的性

**ユーザーの目標が明確でサポートされている**
- テキストはユーザーがタスクを完了するのを助ける
- ユーザーの「なぜここにいるのか？」という質問に答える
- アクションへの障壁を取り除く

**ビジネス目標が達成される**
- コンバージョン、エンゲージメント、またはリテンションをサポート
- プロダクト目標と一致
- 組織目標を前進させる

**ブランドボイスが反映される**
- ブランドパーソナリティと一貫性がある
- このプロダクトから来たものとして認識できる
- コンテキストに適したトーンを使用

**価値提案が明白**
- ユーザーのメリットが明確
- 「私にとって何があるのか？」を説明
- アクションを求める前に価値を示す

**主題はユーザーのメリットに焦点を当てる**
- 二人称（「あなた」）で書かれている
- 機能ではなく成果を強調
- ユーザー中心、会社中心ではない

**能動的で招待的なフレーミング**
- 押し付けがましくなくアクションを動機づける
- ユーザーの意図とジャーニーステージに合っている
- 適切なレベルの緊急性

## 会話性

**自然な話し言葉**
- 声に出して言うようなもの
- 読むときに自然に流れる
- 堅苦しくない、過度にフォーマルでない

**能動態が優勢**
- 主語がアクションを実行
- 直接的でエネルギッシュ
- 受動態は明確な場合のみ使用（まれなケース）

**接続詞を含む**
- 前置詞が存在（「に」「から」「と」）
- 冠詞を含む（「a」「an」「the」）
- 電報的またはロボット的でない

**馴染みのある言葉とフレーズ**
- ユーザーが使う言葉を使用
- ユーザー調査とテストに基づく
- 不要な技術的専門用語なし

**適切な瞬間でのパーソナリティ**
- コンテキストが許すときにボイスが輝く
- 軽いコンテキストで過度に真剣でない
- 真剣な瞬間で遊び心がない

## 明確性

**正確なアクションワード**
- アクションを説明する具体的な動詞
- 永久削除には「削除」であって「除去」ではない
- 変更を保存するには「保存」であって「OK」ではない

**コマンド形式を適切に使用**
- ボタンには能動的な命令形
- 明確で直接的な指示
- アクション指向の言語

**平易な言葉**
- 一般向けには中学1年生の読解レベル
- 専門的なコンテキストでは高校1年生
- 複雑な語彙と文構造を避ける

**意味のある説明的なタイトル**
- タイトルはあなたがどこにいるかを伝える
- 一般的または曖昧でない
- コンテキストと方向性を提供

**一貫したパターンと用語**
- 同じ言葉は全体を通して同じ意味
- UIパターンを一貫して適用
- スタイルガイドに文書化された用語

## スコアリングガイド

**9-10**: 優秀 — ベストプラクティスの例
**7-8**: 良い — わずかな改善が可能
**5-6**: 適切 — 対処すべき顕著な問題
**3-4**: 作業が必要 — 重大な問題
**0-2**: 悪い — 大幅な改訂が必要

## このチェックリストの使い方

1. **個別の要素をレビュー** — 特定のUI文字列、メッセージ、またはフローを評価
2. **エクスペリエンス全体を監査** — 主要な画面またはユーザージャーニーをスコアリング
3. **オプションを比較** — 異なるバージョンを評価して最良のものを選択
4. **改善を追跡** — 編集前後を測定
5. **根拠を構築** — スコアを使用してライティングの決定を説明
6. **努力を集中** — 最初に改善するために最もスコアの低い領域を特定

## 評価例

**テキスト**: 「リクエストの処理中にエラーが発生しました。もう一度お試しください。」

**簡潔性**: 6/10 — 冗長、「リクエストを処理できませんでした。もう一度お試しください。」にできる
**目的性**: 4/10 — ユーザーが問題を修正したり、何が起こったかを説明したりするのに役立たない
**会話性**: 5/10 — やや機械的、「エラーが発生しました」はシステム用語
**明確性**: 5/10 — 曖昧、どのエラーか、なぜかを特定していない

**全体**: 5/10 — 適切だが大幅な改善が必要

**改善版**: 「変更を保存できませんでした。接続を確認してもう一度お試しください。」

**簡潔性**: 9/10 — 簡潔、直接的、無駄な言葉なし
**目的性**: 8/10 — 考えられる原因（接続）と次のステップを提案
**会話性**: 9/10 — 自然で人間的な表現
**明確性**: 9/10 — 何が失敗したか、何をすべきかについて具体的

**全体**: 9/10 — 優秀
