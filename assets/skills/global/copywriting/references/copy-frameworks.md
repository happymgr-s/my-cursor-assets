## ランディングページのセクションタイプ

### コアセクション

**ヒーロー（ファーストビュー）**
- ヘッドライン + サブヘッドライン
- メインCTA
- サポートビジュアル（製品スクリーンショット、ヒーロー画像）
- オプション：ソーシャルプルーフバー

**ソーシャルプルーフバー**
- 顧客ロゴ（多い > 認知度が高い）
- 主要指標（「10,000以上のチーム」）
- 星評価とレビュー数
- 短い証言スニペット

**問題・ペインセクション**
- 顧客本人よりもうまく問題を言語化する
- 共感を生む（「まさに自分の状況だ」）
- 解決しないことのコストをほのめかす

**ソリューション・ベネフィットセクション**
- 問題からソリューションへの橋渡し
- 主要ベネフィット3〜5個（10個ではなく）
- 各ベネフィット：ヘッドライン + 説明 + 可能なら証拠

**使い方（How It Works）**
- 番号付き3〜4ステップ
- 複雑さの印象を軽減する
- 各ステップ：アクション + 成果

**最終CTAセクション**
- バリュープロポジションの要約
- メインCTAの繰り返し
- リスク軽減（保証、無料トライアル）

---

### サポートセクション

**お客様の声（Testimonials）**
- 氏名・役職・会社名入りの引用
- 可能であれば顔写真
- 漠然とした賞賛より具体的な成果
- 形式：引用カード、動画、Twitterの埋め込み

**事例紹介（Case Studies）**
- 問題 → ソリューション → 結果
- 具体的な数値と成果
- 顧客名とコンテキスト
- 「続きを読む」リンク付きのスニペットでも可

**ユースケース**
- 製品のさまざまな使い方
- 訪問者の自己識別を促す
- 「Xが必要なマーケター向け」の形式

**ペルソナ・「〜のために作られた」セクション**
- ターゲットオーディエンスを明示する
- 「[役職]に最適」ブロック
- 「これは自分向け？」という疑問に応える

**FAQセクション**
- よくある異論に対処する
- SEOに有効
- サポートコストを削減する
- よくある質問5〜10個

**比較セクション**
- 競合との比較（名指しするかどうかは任意）
- 現状（スプレッドシート、手作業）との比較
- 表やサイドバイサイド形式

**連携・パートナー**
- 連携するツールのロゴ
- 「あなたのスタックと連携」というメッセージ
- 信頼性を構築する

**創業者ストーリー・マニフェスト**
- なぜ作ったか
- 何を信じているか
- 感情的なつながり
- 顔のない競合との差別化

**デモ・製品ツアー**
- インタラクティブデモ
- 動画ウォークスルー
- GIFプレビュー
- 製品の動作を見せる

**料金プレビュー**
- 料金ページ以外でもティーザーとして見せる
- 「〇〇円〜」という開始価格
- 意思決定者の背中を押す

**保証・リスク軽減**
- 返金保証
- 無料トライアルの条件
- 「いつでも解約可能」
- 摩擦を減らす

**実績セクション**
- 信頼性を構築する主要指標
- 「10,000以上の顧客」
- 「4.9/5の評価」
- 「顧客に2億円の節約」

## ページ構成テンプレート

### 機能羅列型ページ（弱い）

```
1. ヒーロー
2. 機能1
3. 機能2
4. 機能3
5. 機能4
6. CTA
```

これはリストであり、説得力のある物語ではない。

---

### メリハリのある魅力的なページ（強い）

```
1. 明確なバリュープロポジションを持つヒーロー
2. ソーシャルプルーフバー（ロゴまたは統計）
3. 問題・ペインセクション
4. 使い方（3ステップ）
5. 主要ベネフィット（2〜3個、10個ではなく）
6. お客様の声
7. ユースケースまたはペルソナ
8. 代替案との比較
9. 事例紹介スニペット
10. FAQ
11. 保証付き最終CTA
```

ストーリーを語り、異論に対処している。

---

### コンパクトなランディングページ

```
1. ヒーロー（ヘッドライン、サブヘッドライン、CTA、画像）
2. ソーシャルプルーフバー
3. アイコン付き主要ベネフィット3個
4. お客様の声
5. 使い方（3ステップ）
6. 保証付き最終CTA
```

簡潔さが重要な広告ランディングページに適している。

---

### エンタープライズ/B2Bランディングページ

```
1. ヒーロー（成果重視のヘッドライン）
2. ロゴバー（著名企業）
3. 問題セクション（ビジネス上の課題）
4. ソリューション概要
5. 役職・部門別ユースケース
6. セキュリティ・コンプライアンスセクション
7. 連携ロゴ
8. 指標付き事例紹介
9. ROI・価値セクション
10. 問い合わせ・デモCTA
```

エンタープライズバイヤーの懸念に対処している。

---

### 製品ローンチページ

```
1. ローンチ告知付きヒーロー
2. 動画デモまたはウォークスルー
3. 機能ハイライト（3〜5個）
4. ビフォー・アフター比較
5. 初期の証言
6. ローンチ価格またはアーリーアクセスオファー
7. 緊迫感のあるCTA
```

ProductHunt、ローンチ、または告知に適している。

## セクション執筆のヒント

### 問題セクション

以下のような書き出しから始める：
- 「この感覚、わかりますよね...」
- 「多くの[役職]の方と同じように...」
- 「毎日、[オーディエンス]は〜に悩んでいます...」
- 「誰もが経験したことがあるでしょう...」

次に以下を描写する：
- 具体的なフラストレーション
- 無駄にしている時間やお金
- 仕事や生活への影響

### ベネフィットセクション

各ベネフィットに含めること：
- **ヘッドライン**：得られる成果
- **本文**：どのように機能するか（1〜2文）
- **証拠**：数値、証言、または例（オプション）

### 使い方セクション

各ステップは以下のようにする：
- **番号付き**：進捗感を生み出す
- **シンプルな動詞**：「接続する」「設定する」「受け取る」
- **成果指向**：このステップで何が得られるか

例：
1. ツールを接続する（2分で完了）
2. 設定する
3. 毎週月曜日に自動レポートを受け取る

### 証言の選び方

最良の証言に含まれるもの：
- 具体的な成果（「コンバージョンが32%増加」）
- ビフォー・アフターのコンテキスト（「以前は何時間もかかっていました...」）
- 信頼性のための役職と会社名
- 引用に値する具体的な内容

以下のような証言は避ける：
- 「素晴らしい製品！」
- 「大好き！」
- 「使いやすい！」
